## 5. Clarification Step Optimization

Use the cheapest model for the clarification step (question generation), regardless of the compilation model selected:

```typescript
function getCheapModel(config: KairnConfig): { model: string; baseURL?: string } {
  const providerConfig = PROVIDER_CONFIGS[config.provider];
  return {
    model: providerConfig?.cheapModel || config.model,
    baseURL: providerConfig?.baseURL || config.base_url,
  };
}
```

This saves cost — clarification uses Haiku/nano/flash, compilation uses the selected model.

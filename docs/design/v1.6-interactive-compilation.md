# v1.6.0 Design Doc — Interactive Compilation & CLI Polish

> Add a clarification step before compilation to eliminate hallucination, and polish the CLI output with branded colors and structured layouts.

**Status:** Planned  
**Depends on:** v1.5.0

---

## 1. Interactive Clarification Flow

### Problem
When users give vague intents like `kairn describe "a cli tool for agentic infrastructure"`, the LLM invents project details (frameworks, features, architecture) that the user never specified. This produces hallucinated CLAUDE.md content.

### Solution
Add a clarification step between intent capture and compilation. The LLM generates 3-5 clarifying questions with recommended defaults. The user can accept defaults (press Enter) or type their own answer.

### UX Flow
```
$ kairn describe "a cli tool for agentic infrastructure"

  ┌─────────────────────────────────────────────────┐
  │  KAIRN — Environment Compiler                   │
  └─────────────────────────────────────────────────┘

  Let me understand your project better.
  Press Enter to accept the suggestion, or type your own answer.

  Language/framework?
  › TypeScript + Node.js (suggested)

  What does it do specifically?
  › Provisions and orchestrates AI agent runtimes (suggested)

  Primary workflow?
  › Build CLI commands, write tests, ship to npm (suggested)

  Key dependencies or integrations?
  › None specified (suggested)

  Who uses this?
  › Developers building agent systems (suggested)

  ──────────────────────────────────────────────────

  Compiling with anthropic (claude-sonnet-4-6)...
  Parsing environment spec...

  ✓ Environment compiled
  ...
```

### Implementation

#### Step 1: Pre-compilation LLM call for questions
Add a new function `generateClarifications()` in `src/compiler/compile.ts`:
- Makes a cheap, fast LLM call (Haiku) with the user's raw intent
- System prompt instructs: "Generate 3-5 clarifying questions with suggested answers. Output as JSON array."
- Returns: `[{ question: string, suggestion: string }]`

```typescript
interface Clarification {
  question: string;
  suggestion: string;
}

async function generateClarifications(
  intent: string,
  config: KairnConfig
): Promise<Clarification[]> {
  // Use Haiku for speed/cost — this is a simple task
  const response = await callLLM(config, clarificationPrompt + intent, "haiku");
  return JSON.parse(response);
}
```

#### Step 2: Clarification system prompt
Add to `src/compiler/prompt.ts`:

```typescript
export const CLARIFICATION_PROMPT = `You are helping a user define their project for environment compilation.

Given their initial description, generate 3-5 clarifying questions to understand:
1. Language and framework
2. What the project specifically does (be precise)
3. Primary workflow (build, research, write, analyze?)
4. Key dependencies or integrations
5. Target audience

For each question, provide a reasonable suggestion based on the description.

Output ONLY a JSON array:
[
  { "question": "Language/framework?", "suggestion": "TypeScript + Node.js" },
  ...
]

Rules:
- Suggestions should be reasonable guesses, clearly marked as suggestions
- Keep questions short (under 10 words)
- Maximum 5 questions
- If the description is already very detailed, ask fewer questions`;
```

#### Step 3: Interactive prompts in describe.ts
Update `src/commands/describe.ts` to:
1. Call `generateClarifications()` with raw intent
2. Display each question with the suggestion as default
3. Collect answers (Enter = accept suggestion, or type custom)
4. Build enriched intent string from answers
5. Pass enriched intent to `compile()`

```typescript
// For each clarification:
const answer = await input({
  message: clarification.question,
  default: clarification.suggestion,
});
```

#### Step 4: Enriched intent format
The enriched intent passed to the compiler becomes:

```
User intent: "a cli tool for agentic infrastructure"

Clarifications:
- Language/framework: TypeScript + Node.js
- What it does: Provisions and orchestrates AI agent runtimes
- Primary workflow: Build CLI commands, write tests, ship to npm
- Key dependencies: None specified
- Target audience: Developers building agent systems
```

This gives the compiler real information instead of forcing it to guess.

#### Step 5: Skip clarification with --quick flag
Add `--quick` or `-q` flag to skip clarifications and compile directly (old behavior):
```bash
kairn describe "build a next.js app" --quick  # no questions, straight to compile
```

The existing `--yes` flag skips the write confirmation but NOT clarifications.

## 2. CLI Visual Design

### Problem
Current output is functional but plain. No brand identity. Hard to scan for important information in longer outputs.

### Solution
Establish a branded color palette and consistent output structure across all commands.

### ASCII Cairn Logo
Display the cairn logo in maroon at the top of key commands (describe, optimize, init):

```
       ╭─╮
       ╰┬╯
      ╭─┴─╮
      ╰─┬─╯
     ╭──┴──╮
     ╰──┬──╯
    ╭───┴───╮
    │ KAIRN │
    ╰───────╯
```

Implementation in `src/ui.ts`:
```typescript
export function logo(): string {
  const m = chalk.rgb(139, 0, 0);  // maroon
  return [
    m("       ╭─╮"),
    m("       ╰┬╯"),
    m("      ╭─┴─╮"),
    m("      ╰─┬─╯"),
    m("     ╭──┴──╮"),
    m("     ╰──┬──╯"),
    m("    ╭───┴───╮"),
    m("    │") + chalk.bold.rgb(139, 0, 0)(" KAIRN ") + m("│"),
    m("    ╰───────╯"),
  ].map(l => "  " + l).join("\n");
}
```

### Color Palette
```
BRAND:
  Maroon/Dark Red:  #8B0000 → chalk.rgb(139, 0, 0)     — logo, headers, brand elements
  Warm Accent:      #D4A574 → chalk.rgb(212, 165, 116)  — highlights, bullets, links

STATUS:
  Success Green:    chalk.green                          — ✓ confirmations
  Warning Yellow:   chalk.yellow                         — ⚠ warnings, API keys needed
  Error Red:        chalk.red                            — ✗ errors

INFO:
  Muted/Dim:        chalk.dim                            — file paths, descriptions
  Cyan:             chalk.cyan                           — labels, section headers
  Bold:             chalk.bold                           — values, commands
```

### Implementation

#### Create `src/ui.ts` — centralized styling functions

```typescript
import chalk from "chalk";

// Brand colors
const maroon = chalk.rgb(139, 0, 0);
const warm = chalk.rgb(212, 165, 116);

export const ui = {
  // Brand
  brand: (text: string) => maroon.bold(text),
  accent: (text: string) => warm(text),

  // Headers
  header: (text: string) => {
    const line = "─".repeat(50);
    return `\n  ${maroon("┌" + line + "┐")}\n  ${maroon("│")} ${maroon.bold(text.padEnd(49))}${maroon("│")}\n  ${maroon("└" + line + "┘")}\n`;
  },

  // Sections
  section: (title: string) => `\n  ${warm("━━")} ${chalk.bold(title)} ${warm("━".repeat(Math.max(0, 44 - title.length)))}`,

  // Status
  success: (text: string) => chalk.green(`  ✓ ${text}`),
  warn: (text: string) => chalk.yellow(`  ⚠ ${text}`),
  error: (text: string) => chalk.red(`  ✗ ${text}`),
  info: (text: string) => chalk.cyan(`  ℹ ${text}`),

  // Key-value pairs
  kv: (key: string, value: string) => `  ${chalk.cyan(key.padEnd(14))} ${value}`,

  // File list
  file: (path: string) => chalk.dim(`    ${path}`),

  // Tool display
  tool: (name: string, reason: string) => `    ${warm("●")} ${chalk.bold(name)}\n      ${chalk.dim(reason)}`,

  // Divider
  divider: () => chalk.dim(`  ${"─".repeat(50)}`),

  // Command suggestion
  cmd: (command: string) => `    ${chalk.bold.white("$ " + command)}`,

  // Env var setup
  envVar: (name: string, desc: string, url?: string) => {
    let out = `    ${chalk.bold(`export ${name}=`)}${chalk.dim('"your-key-here"')}\n`;
    out += chalk.dim(`      ${desc}`);
    if (url) out += `\n      ${chalk.dim("Get one at:")} ${warm(url)}`;
    return out;
  },

  // Clarification question
  question: (q: string, suggestion: string) =>
    `  ${warm("?")} ${chalk.bold(q)}\n    ${chalk.dim(`suggested: ${suggestion}`)}`,
};
```

## 3. Additional CLI Improvements

### Progress spinner
Replace the current `process.stdout.write` line-overwrite with a proper spinner using the `ora` package (lightweight, widely used):

```bash
npm install ora
```

```typescript
import ora from "ora";

const spinner = ora("Compiling environment...").start();
// ... LLM call
spinner.succeed("Environment compiled");
```

### `--no-color` flag
Add global `--no-color` flag that disables all chalk styling for piping/CI:
```bash
kairn describe "..." --no-color > output.txt
```

### Better error display
Errors should use the branded error style:
```
  ┌──────────────────────────────────────────────────┐
  │ KAIRN — Error                                    │
  └──────────────────────────────────────────────────┘

  ✗ Compilation failed

  Rate limited by Anthropic. Wait a moment and try again,
  or switch to a cheaper model with `kairn init`.
```

   - Press Enter on all — verify suggestions are used in compilation
   - Type custom answers — verify they override suggestions
   - Run `kairn describe "Build a Next.js 15 app with Supabase auth and Stripe payments" --quick` — verify no questions asked
